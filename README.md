# Petstore Postman Tests

## Опис
Колекція Postman для тестування CRUD операцій з сутністю Pet
використовуючи https://petstore.swagger.io

## Структура
- Create Pet
- Get Pet
- Update Pet
- Get Updated Pet

## Запуск
1. Імпортувати колекцію в Postman
2. Запустити через Collection Runner
3. Environment не потрібен (використовуються dynamic variables)

## Тести
Кожен запит містить автоматизовані тести:
- HTTP status code
- Валідація тіла відповіді
- Перевірка бізнес-логіки

## Завдання 2
Описано 10 тест-кейсів для ендпоінта POST /pet
Тест-кейси:
1. Створення пета з валідними даними → 200 OK
2. Перевірка, що id у відповіді відповідає запиту
3. Створення пета без name → error / validation
4. Створення пета з існуючим id
5. Перевірка типу поля status
6. Перевірка Content-Type = application/json
7. Створення пета з великим числом id
8. Створення пета з довгим name (>255)
9. Перевірка часу відповіді (<2s)
10. Перевірка структури відповіді (id, name, status)
